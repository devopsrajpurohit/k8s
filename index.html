CI/CD

Best Practices for CI/CD

  • Use GitOps tools like Argo CD or Flux for declarative deployments.
  • Automate builds via GitHub Actions, Jenkins, or GitLab CI with SBOM generation and signing.
  • Adopt health checks and auto‑rollbacks to ensure deployment safety.
  • Separate application and environment configuration repositories.
# Example GitHub Action Workflow
name: Build and Push Image
on: [push]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: docker build -t ghcr.io/org/app:${{ github.sha }} .
      - run: echo $CR_PAT | docker login ghcr.io -u $GITHUB_ACTOR --password-stdin
      - run: docker push ghcr.io/org/app:${{ github.sha }}
Security

Best Practices for Security

  • Apply Pod Security (restricted) and use non‑root containers.
  • Enforce RBAC and limit admin privileges.
  • Use NetworkPolicies for least privilege network access.
  • Continuously scan images using Trivy or OPA Gatekeeper.
# Example NetworkPolicy
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default-deny-egress
spec:
  podSelector: {}
  policyTypes: [Egress]
  egress:
  - to:
    - ipBlock: { cidr: 10.0.0.0/16 }
Cost

Best Practices for Cost Optimization

  • Right‑size pods using Vertical and Horizontal Pod Autoscalers.
  • Use Kubecost/OpenCost to monitor per‑namespace spend.
  • Clean unused resources like PVCs, Jobs, and images regularly.
  • Leverage Cluster Autoscaler and Karpenter for elasticity.
# Example HPA
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: webapp-hpa
spec:
  minReplicas: 2
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
Reliability

Best Practices for Reliability

  • Use PodDisruptionBudgets to maintain availability during updates.
  • Distribute workloads across zones using topologySpreadConstraints.
  • Maintain runbooks and auto‑remediation workflows for recurring issues.
  • Perform regular backups using Velero.
# Example PodDisruptionBudget
apiVersion: policy/v1
kind: PodDisruptionBudget
metadata:
  name: webapp-pdb
spec:
  minAvailable: 2
  selector:
    matchLabels:
      app: webapp
Monitoring

Best Practices for Monitoring

  • Set up Prometheus, Grafana, and Loki for observability.
  • Define SLOs and alerting rules using burn rate calculations.
  • Maintain good labeling for service, team, severity, and runbook links.
  • Integrate alerts with your on‑call platform for instant escalation.
# Example Prometheus Alert
- alert: HighErrorRate
  expr: rate(http_requests_total{status=~"5.."}[5m]) / rate(http_requests_total[5m]) > 0.05
  for: 10m
  labels:
    severity: critical
    service: api
  annotations:
    summary: "5xx > 5% for 10m"
